INSTALL notes for asdlGen version 1.0b

Daniel C. Wang

January 30, 1998

1. The Binary Distribution

Since installing the distribution from sources requires the SML/NJ compiler, I'm providing a binary distribution for those who just want to use asdlGen in a development environment that does not have SML/NJ installed. Fetch the right binary version from http://www.cs.princeton.edu/zephyr/ASDL/#software.

1.1 Unix

On a Unix system with ksh all you should have to do is unpack the archive and put the shell script bin/asdlGen in your path. If your stuck on a Unix system without ksh you'll have to hack the script so it can locate the directory where you unpacked the sources.

1.2 Windows NT/95

In a Windows environment you need to set the environment variable ASDL_HOME to the location of where you unpacked the distribution. Also set your path to include bin/asdlGen.bat.

2. Installing from Sources

asdlGen now builds with autoconf and automake. To build from sources you should have the latest version 110.0.2 of Standard ML of New Jersey. Binary release are also available from http://www.cs.princeton.edu/zephyr/ASDL/#software. I plan to experiment with releasing sources via a read only remote CVS repository.

2.1 Unix

On a Unix machine run

./configure --prefix=Your install prefix
let configure do it's thing, and do a make install

2.2 Windows NT/95

I haven't figured out how to automatically compile the supporting Java, C, and C++ libraries, but here are instructions on how to build the asdlGen program itself.

On a Windows machine after sml-cm starts up in src/sml/. 

Standard ML of New Jersey, Version 110.0.2, January 16, 1998 
- use "build.sml";
